## 📦 자원 명세

### 컴퓨트

| 자원 | 스펙 | 리전 | 용도 | 월 비용(USD) |
|------|------|------|------|-------------|
| **EC2** | `t3.medium` (2vCPU, 4GB RAM) | `ap-northeast-2` (서울) | Docker Compose (web + api + nginx) | ~$30 |
| **EBS** | `gp3` 30GB | 서울 | EC2 루트 볼륨 | ~$3 |
| **ECR** | Private repo (web, api 각 1개) | 서울 | Docker 이미지 저장소 | ~$0.5 |

### 데이터

| 자원 | 스펙 | 리전 | 용도 | 월 비용(USD) |
|------|------|------|------|-------------|
| **RDS** | `db.t4g.small` (2vCPU, 2GB), Single-AZ | 서울 | PostgreSQL 16 | ~$30 |
| **RDS Storage** | `gp3` 20GB | 서울 | DB 스토리지 | ~$3 |
| **S3** | Standard 클래스 | 서울 | 식당/메뉴 이미지 | ~$1-3 |

### 네트워크/보안

| 자원 | 스펙 | 용도 | 월 비용(USD) |
|------|------|------|-------------|
| **CloudFront** | 기본 가격 정책 | 정적/이미지 CDN, SSL 종단 | ~$3-5 |
| **Route 53** | Hosted Zone 1개 | DNS | ~$1 |
| **ACM** | DV 인증서 | HTTPS (CloudFront 용 → **`us-east-1` 리전 발급 필수**) | $0 |
| **데이터 전송** | EC2 → 인터넷 | API/HTML 응답 | ~$5-10 |

### 관측

| 자원 | 스펙 | 용도 | 월 비용(USD) |
|------|------|------|-------------|
| **CloudWatch Logs** | Docker `awslogs` driver | 컨테이너 로그 수집 | ~$2-5 |
| **CloudWatch Alarms** | EC2 CPU/메모리, RDS CPU/Connection 임계 알람 | 장애 알림 | ~$0-2 |

### 합계

**월 ~$80~90** (S3/CDN 트래픽에 따라 변동)

## 🔐 보안 / 권한

### IAM

- **API 서버용 IAM 사용자**: S3 PutObject, GetObject 만 허용 (특정 prefix 한정)
- **GitHub Actions 용 IAM 사용자**: ECR push, SSM Run Command 권한
- 루트 계정으로 작업 ❌, 항상 IAM 사용자/역할 사용

### 시크릿

- 환경변수는 EC2 안 `.env.production` 파일에 저장 (660 권한)
- 또는 AWS Secrets Manager 사용 (선택, 월 $0.4/시크릿)
- GitHub Secrets: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `EC2_INSTANCE_ID` 등

### 보안 그룹

| SG | 인바운드 | 비고 |
|----|---------|------|
| `pangchelin-alb-sg` (옵션) | 80, 443 from 0.0.0.0/0 | ALB 사용 시에만 |
| `pangchelin-ec2-sg` | 80, 443 from 0.0.0.0/0 (또는 CloudFront IP) | SSM 사용 시 SSH(22) 불필요 |
| `pangchelin-rds-sg` | 5432 **from `pangchelin-ec2-sg` 만** | 외부 직접 접근 차단 |

## 🌐 도메인 / SSL

- 도메인: 미정 (예: `kwueats` 등)
- SSL: ACM 무료 인증서, **CloudFront 용은 us-east-1 에서 발급**
- DNS: Route 53 ALIAS 레코드로 CloudFront 분배 도메인 가리킴

---

## 🚀 배포 / CI-CD

- **이미지 빌드**: GitHub Actions → Docker build → ECR push (태그: git SHA)
- **배포 트리거**: main 브랜치 머지 시 SSM Run Command 로 EC2 에서 `docker compose pull && up -d --remove-orphans`
- **PR 검증**: lint, build, Docker build 만 (push 안 함)
- **롤백**: ECR 의 이전 git SHA 태그로 docker-compose 의 image 만 되돌리고 재배포

## 📊 모니터링 / 알람

CloudWatch 에서 다음 알람 설정:

- EC2 CPU > 70% 5분 이상
- EC2 메모리 > 85% 5분 이상 (CloudWatch Agent 설치 필요)
- EC2 CPU credit balance < 100 (버스트 한계 임박)
- RDS CPU > 70%
- RDS FreeableMemory < 200MB
- RDS DatabaseConnections > 60 (max 83)

알람 채널: 이메일 (SNS) 또는 Slack Webhook

## 💾 백업 / 복구

| 대상 | 방식 | 보관 |
|------|------|------|
| **RDS** | 자동 백업 (PITR) | 7~14일 |
| **RDS** | 수동 스냅샷 | 주요 변경 직전 |
| **S3** | 버저닝 + Lifecycle (30일 이전 버전 삭제) | 무제한 |
| **EC2 루트 EBS** | AWS Backup 또는 cron 스냅샷 | 7일 |

복구 시나리오 별 RTO/RPO:

- DB 사고: PITR 5분 RPO, 30분 RTO
- EC2 사고: AMI 또는 docker-compose 재배포 30분 RTO
- 이미지 사고: S3 버저닝으로 즉시 복원

## 📈 확장 시나리오

| 트래픽 | 조치 |
|--------|------|
| 100~500 CCU (현재 목표) | 단일 EC2 + RDS Single-AZ |
| 500~1000 CCU | EC2 t3.large 1대 + Redis (ElastiCache) 추가 |
| 1000+ CCU | ALB + Auto Scaling Group (EC2 2대~) + RDS Read Replica |
